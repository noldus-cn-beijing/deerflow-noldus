# EthoInsight 用户使用旅程图

> **版本**: V1 | **日期**: 2026-04-14 | **范围**: 软件内操作流程 | **状态**: 草案

---

## 旅程概述

**目标用户**：研究生/实验技术员（主要）、行为学研究员/博后（次要）
**采购决策者**：实验室负责人/PI
**旅程范围**：从打开软件到导出报告的完整操作流程
**目标**：将数据分析时间从 4-5 小时压缩至 5 分钟以内

---

## 用户画像对照

| 维度 | 小王（研究生/技术员） | 李同学（高年级博后） |
|------|----------------------|---------------------|
| 统计基础 | 知道 t 检验概念，不确定何时用哪个 | 能独立选择统计方法 |
| 行为学知识 | 了解正在做的范式基本原理 | 对研究范式有深入理解 |
| 核心诉求 | "告诉我结果就行" | "帮我快速出初稿，我要能调参数" |
| 首选模式 | 全自动模式 | 手动模式 |
| 情绪基线 | 焦虑——怕选错方法、怕解读偏差 | 不耐烦——想要高效、想要掌控 |

---

## 旅程总览

```
启动准备 → 数据导入 → 解析验证 → 模式选择 → 分析配置 → 计算执行 → 报告审阅 → 追问对话 → 导出分享
  (1min)    (1min)     (30s)      (10s)     (1-3min)    (30s)     (1min)    (开放)     (30s)
```

**全自动模式总耗时**：约 3-4 分钟
**手动模式总耗时**：约 5-8 分钟（取决于用户决策速度）

## 阶段 1：启动与准备

> 用户刚跑完实验，打开 EthoInsight，准备分析数据。

### 用户行为
- 打开 EthoInsight（桌面应用 / Web 端）
- 看到主界面，了解可选操作
- 准备好 EthoVision 导出的 CSV/Excel 文件

### 触点
- 应用启动页/主界面
- 新建分析入口

### 用户情绪

| 小王 | 李同学 |
|------|--------|
| 期待但紧张——"希望这次能比上次手动分析轻松" | 急迫——"还有三组数据等着处理" |
| 不确定——"我该从哪里开始？" | 熟练——"直接上传就行" |

### 痛点
- **首次使用障碍**：不清楚产品能做什么、不能做什么
- **数据准备焦虑**：不确定导出格式是否正确

### 成功指标
- 首次启动到开始操作的引导完成率
- 主界面任务入口的点击热图分布

### 改进机会
- 提供首次使用引导（交互式教程或高亮提示）
- 在上传入口旁展示支持的数据格式示例
- 显示"最近分析"历史，方便重复使用

---

## 阶段 2：数据导入

> 用户上传 EthoVision 导出的数据文件。

### 用户行为
- 点击"上传数据"或拖拽文件到上传区域
- 选择 CSV 或 Excel 文件
- 等待上传完成

### 触点
- 文件上传组件（拖拽 / 点击选择）
- 上传进度指示器
- 上传成功/失败反馈

### 用户情绪

| 小王 | 李同学 |
|------|--------|
| 不确定——"这个格式对不对？" | 习以为常——"拖进去就行" |
| 担忧——"数据会不会弄丢？" | 期待——"看看识别得准不准" |

### 痛点
- **格式不确定**：EthoVision 导出的文件有多种格式变体，用户不确定哪种被支持
- **大文件等待**：数据量大时上传等待焦虑
- **上传失败无解**：上传失败时不知道问题出在哪里

### 成功指标
- 上传成功率（目标 > 95%）
- 上传到识别的等待时间（目标 < 10s）
- 上传失败后的重试成功率

### 改进机会
- 支持 EthoVision 所有常见导出格式变体（.csv, .xlsx, .xls）
- 文件上传前进行格式预检，提前告知兼容性
- 上传失败时给出具体原因和修复建议（如"检测到编码问题，建议另存为 UTF-8"）
- 支持批量上传（同时上传多组数据）

---

## 阶段 3：数据解析与验证

> AI 自动识别实验范式类型、解析数据结构、执行数据质量检查。这是"AI 魔法"第一次显现。

### 用户行为
- 查看系统自动识别的实验范式（如"检测到：高架十字迷宫 EPM"）
- 确认或修正范式识别结果
- 查看数据质量报告（缺失值、异常值、样本量评估）
- 确认变量映射关系（原始列名 → 标准指标名）

### 触点
- 范式识别结果展示卡片
- 数据质量摘要面板
- 变量映射预览表
- 异常值/缺失值高亮显示

### 用户情绪

| 小王 | 李同学 |
|------|--------|
| 惊喜——"它居然知道这是 EPM！" | 审视——"让我看看识别对不对" |
| 安心——"数据质量检查帮我排雷了" | 期待——"希望映射准确，省得我手动配" |
| 困惑（如有异常）——"这个缺失值怎么办？" | 警觉（如有异常）——"这组数据质量不太好" |

### 痛点
- **范式误识别**：列名不规范时可能识别错误，用户可能不会注意到
- **异常值处理迷茫**：检测到异常值后，用户不知道该排除还是保留
- **变量映射不透明**：用户不理解"openArmTime"是怎么被映射到"开臂时间"的

### 成功指标
- 范式自动识别准确率（目标 > 95%）
- 数据质量问题检出率
- 用户在此步骤的平均停留时间（目标 < 1 分钟）

### 改进机会
- 识别结果附带置信度，低置信度时主动提示用户确认
- 提供异常值的上下文信息（该值在整体分布中的位置），辅助用户决策
- 变量映射可视化——用连线或表格清晰展示"你的列名→标准指标"的映射
- 当检测到数据质量问题时，提供具体建议而非仅仅标记

---

## 阶段 4：模式选择

> 用户选择全自动模式或手动模式。这是"手自一体"设计的核心分叉点。

### 用户行为
- 系统展示模式选择界面
- **全自动模式**：一键开始，系统自动完成所有分析
- **手动模式**：进入指标选择、统计方法选择、图表类型选择的配置流程
- 首次使用用户可能需要帮助理解两种模式的区别

### 触点
- 模式选择界面（卡片/切换按钮）
- 模式说明提示（Tooltip 或简短描述）

### 用户情绪

| 小王 | 李同学 |
|------|--------|
| 犹豫——"全自动靠谱吗？" | 果断——"手动模式，我要选参数" |
| 安心（选择全自动后）——"不用我选，太好了" | 掌控感（选择手动后）——"我想用 Mann-Whitney" |

### 痛点
- **选择困难**：新手不知道该选哪个模式，担心全自动不靠谱，又怕手动选错
- **模式切换成本**：选了全自动后想调参数，或者选了手动后觉得太复杂

### 成功指标
- 全自动模式选择率（预期 70-80%）
- 手动模式选择率（预期 20-30%）
- 模式切换率（分析中途从手动切到全自动，或反之）

### 改进机会
- 默认推荐全自动模式（对新手友好），手动模式作为"高级选项"
- 提供模式预览——"全自动模式将自动选择以下指标和方法：[预览]"
- 支持中途无缝切换——全自动报告出来后可切换到手动模式调整参数
- 记住用户上次选择的模式偏好

---

## 阶段 5：分析配置（手动模式专属）

> 手动模式用户在此步骤选择指标、统计方法和图表类型。全自动模式跳过此步骤。

### 用户行为
- 从系统展示的可选指标列表中勾选需要的指标
- 选择统计方法（系统附带推荐和推荐原因）
- 选择图表类型（柱状图+散点、箱线图等）
- 可选：查看每个选项的简要说明

### 触点
- 指标选择列表（带推荐标签和文献参考）
- 统计方法选择器（附带推荐标注和原因说明）
- 图表类型选择器（附带预览缩略图）
- 帮助提示/信息图标

### 用户情绪

| 小王（如进入手动模式） | 李同学 |
|------|--------|
| 压力——"这么多选项，我不知道选哪个" | 专注——"我要选这几个指标，用非参数检验" |
| 依赖推荐——"它推荐什么我就选什么" | 审视推荐——"为什么推荐 t 检验而不是 ANOVA？" |
| 安心——"系统给了推荐，应该不会太离谱" | 满意——"推荐的确实是我想要的" |

### 痛点
- **选项过载**：不熟悉统计的研究生面对多个检验方法和指标选择感到不知所措
- **推荐不信任**：有经验的研究员可能质疑系统推荐的依据
- **选择后悔**：选完之后不确定是否选对了

### 成功指标
- 手动模式用户接受系统推荐的比例
- 配置完成到开始分析的时间（目标 < 3 分钟）
- 配置过程中打开帮助提示的频率

### 改进机会
- 用"推荐标签 + 推荐原因"降低决策负担（如"推荐 Mann-Whitney U 检验——因为您的数据样本量较小且正态性检验未通过"）
- 提供"一键采用推荐配置"快捷按钮
- 图表类型附带小型预览图，直观展示效果
- 高亮指标之间的关联（如选了开臂时间%，自动建议也选总进臂次数用于运动混杂检查）

---

## 阶段 6：计算执行

> 确定性 Python 脚本执行所有数值计算。AI 不参与数值运算。

### 用户行为
- 等待计算完成（全自动模式 < 30s，手动模式视复杂度而定）
- 观察计算进度指示器
- 可能查看实时计算状态（正在执行正态性检验 → 正在选择统计方法 → 正在计算效应量）

### 触点
- 进度条/步骤指示器
- 计算状态实时反馈
- 透明度面板（可选展开，查看每步的方法选择依据和计算过程）

### 用户情绪

| 小王 | 李同学 |
|------|--------|
| 期待——"快好了吗？" | 期待——"看看结果怎么样" |
| 好奇——"它在做什么？" | 审视——"用的是 Shapiro-Wilk 吗？" |
| 焦虑（如果太慢）——"是不是出问题了？" | 不耐烦（如果太慢）——"怎么这么久？" |

### 痛点
- **黑箱焦虑**：用户不知道系统在做什么，等待时容易产生不安
- **等待时间不确定**：没有明确的预计完成时间

### 成功指标
- 计算完成时间（目标 < 30s for 全自动模式）
- 透明度面板的展开率（了解用户对过程的关注程度）
- 计算成功率（目标 > 99%）

### 改进机会
- 分步展示计算进度（"正态性检验 ✓ → 统计检验 ✓ → 效应量计算中..."）
- 提供"透明度面板"——展示每一步的方法选择依据（如"基于 Shapiro-Wilk 检验 p=0.03 < 0.05，选择非参数检验"）
- 预估剩余时间
- 计算完成后提供简短的"分析摘要"预览（如"检测到 2 组间显著差异，效应量中等"）

---

## 阶段 7：报告审阅

> 系统生成完整的分析报告，包含自然语言解读、统计图表、指标方向解读和综合结论。

### 用户行为
- 阅读分析报告摘要
- 查看统计图表（柱状图+散点、箱线图等）
- 查看指标解读（组间比较结果与行为学意义）
- 查看综合结论和置信度
- 检查运动混杂检查结果

### 触点
- 分析报告页面（结构化展示）
- 统计图表区域（可放大、可切换图表类型）
- 指标解读卡片（阈值参照、文献引用）
- 综合结论区域
- 脚本版本号和执行日志（可展开）

### 用户情绪

| 小王 | 李同学 |
|------|--------|
| 如释重负——"终于有结果了！" | 验证——"结果跟我想的一致吗？" |
| 学习——"原来开臂时间 12% 在这个品系里算偏低的" | 审查——"图表样式需要调整" |
| 信任（如果报告清晰）——"比我自己做的靠谱多了" | 满意/挑剔——"结论够严谨，但图表配色想换" |

### 痛点
- **结论信任度**：新手可能不理解结论的依据，盲目接受或盲目怀疑
- **图表不可调**：报告生成的图表无法微调样式（颜色、字体大小等）
- **运动混杂忽视**：用户可能忽略运动混杂检查的警告，导致错误解读
- **过度解读风险**：结论中"数据支持的发现"和"需要验证的假设"区分不够明显

### 成功指标
- 报告完整度评分（是否包含所有必要元素）
- 用户在报告页面的平均停留时间
- 图表导出率（用户将图表单独导出的比例）
- 透明度面板查看率（用户展开查看计算依据的比例）

### 改进机会
- 结论分层展示：先给"一句话结论"，再展开详细分析
- 运动混杂警告视觉突出（醒目标记，而不仅仅是文字说明）
- 图表支持基本样式调整（颜色、标签位置等）
- 报告中嵌入"可信度说明"——解释结论的依据和数据质量对结论的影响
- 提供"给 PI 看的摘要版"和"完整详细版"两种视图

---

## 阶段 8：追问对话

> 用户基于报告进行多轮追问，深入理解特定结果或调整分析。

### 用户行为
- 点击"追问"或直接在对话框中输入问题
- 典型追问场景：
  - "第 3 组为什么异常？"
  - "能换成 Mann-Whitney 检验吗？"
  - "帮我看看开臂时间趋势"
  - "这个结果的效应量大吗？"
  - "能换个箱线图吗？"
- 查看追问产生的补充分析结果
- 继续追问或结束对话

### 触点
- 追问对话框（聊天式界面）
- 补充分析结果展示区
- 对话历史记录

### 用户情绪

| 小王 | 李同学 |
|------|--------|
| 好奇——"为什么第 3 组异常？" | 探索——"换一种检验试试" |
| 学习——"原来还需要排除运动因素" | 高效——"不用重新跑一遍，直接问就行" |
| 可能困惑——"追问的结果和报告不一致怎么办？" | 审视——"追问产生的分析也是确定性计算吗？" |

### 痛点
- **追问边界不清**：用户不知道可以问什么、不可以问什么
- **对话结果与报告的关系**：追问修改了某些分析后，原始报告是否同步更新？
- **追问质量**：用户不知道怎么描述问题才能得到有用的回答

### 成功指标
- 追问使用率（使用追问功能的用户占比）
- 平均每份报告的追问轮次
- 追问后用户满意度（追问是否解答了用户的问题）
- 对话历史导出率

### 改进机会
- 提供追问建议（如"您可能还想了解：各组的具体指标值"、"查看效应量详情"）
- 追问产生的分析变化自动同步到报告
- 提供追问模板（常见问题快捷入口）
- 明确标注追问产生的分析仍由确定性脚本完成

---

## 阶段 9：导出与分享

> 用户将分析报告导出为可分享的文件格式。

### 用户行为
- 选择导出格式（PDF / Word）
- 可选：选择导出范围（完整报告 / 仅图表 / 仅统计结果）
- 可选：选择报告语言（中文 / 英文）
- 下载导出文件
- 分享给 PI 或用于论文撰写

### 触点
- 导出按钮/菜单
- 导出选项面板（格式、范围、语言）
- 下载进度/完成提示

### 用户情绪

| 小王 | 李同学 |
|------|--------|
| 成就——"终于搞定了，可以交给导师了" | 务实——"导出 Word，我要改几个地方" |
| 自信——"报告看起来很专业" | 期待——"这个报告质量够论文用吗？" |
| 放松——"下次还用这个" | 精益求精——"图表分辨率够高吗？" |

### 痛点
- **格式限制**：只支持 PDF/Word，可能需要 PPT 或 LaTeX 格式
- **编辑困难**：PDF 无法编辑，Word 格式可能不理想
- **语言切换不便**：中英文切换后部分术语翻译不一致
- **图表分辨率担忧**：不确定导出的图表是否达到出版标准

### 成功指标
- 报告导出成功率
- 导出格式偏好分布（PDF vs Word）
- 用户对导出报告质量的满意度评分

### 改进机会
- 支持导出原始数据 + 分析脚本（便于用户复现和审计）
- 提供图表单独导出（PNG/SVG，≥ 300 DPI）
- Word 导出时保留可编辑图表
- 支持报告模板选择（论文格式 / 内部报告格式 / PI 汇报格式）
- 导出前预览，让用户确认最终效果

## 情绪旅程曲线

```
情绪
 ↑
 兴奋  |        ★惊喜                                              ★成就
       |            \                                          /
 安心  |              \            ★安心        ★掌控       /
       |                \          /  \        /    \     /
 中性  |  期待           \        /    \      /      \   /
       | /    \           \      /      \    /        \ /
 焦虑  |/      \           \    /        \  /          ★学习
       |        \紧张       \  / 焦虑等待  \/               \
 不安  |          ‾‾‾‾‾‾‾‾‾‾‾\/‾‾‾‾‾‾‾‾‾‾‾‾
       |
       +————————————————————————————————————————————————————→ 阶段
         启动  导入  解析  模式  配置  计算  审阅  追问  导出
```

**关键情绪拐点：**
1. **正面拐点**（阶段 3→4）：范式自动识别成功时，用户产生"惊喜+信任"
2. **负面拐点**（阶段 6）：计算等待时的焦虑，需用透明度面板缓解
3. **高潮点**（阶段 7）：报告生成后的"如释重负+成就"感
4. **价值确认点**（阶段 9）：导出报告时的"下次还用这个"意愿

| 维度 | 全自动模式（小王） | 手动模式（李同学） |
|------|-------------------|-------------------|
| 总耗时 | 3-4 分钟 | 5-8 分钟 |

| 回访驱动力 | "不用再怕数据分析了" | "比手动快多了" |

## 关键设计建议

### 1. 首次体验是成败关键
- **5 分钟内必须让用户看到第一份报告**——这是建立信任的起点
- 首次使用引导不超过 3 步，每步不超过 10 秒
- 提供"示例数据"一键体验，让用户在上传自己的数据前先感受产品能力

### 2. 透明度贯穿全程
- 每个阶段都有"为什么这样做"的可选解释
- 透明度面板展示 AI 决策依据和确定性计算的执行过程
- 脚本版本号和执行日志始终可查

### 3. 模式无缝切换
- 全自动报告出来后可切换到手动模式调整
- 手动模式可随时"一键采纳推荐"回到全自动路径
- 不需要重新上传数据即可切换模式

### 4. 面向 PI 的导出设计
- 导出报告自动包含方法论说明和文献引用
- 图表达到出版标准（≥ 300 DPI，色盲友好）
- 提供"PI 摘要页"——一页纸概述实验设计、关键发现和结论
