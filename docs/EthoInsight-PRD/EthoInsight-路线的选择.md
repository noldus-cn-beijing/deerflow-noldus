# 我们究竟是要用Result？还是用Data
为防止歧义，下文将软件中的 Statistics Results/统计数据称为`Result`， Raw Data/原始数据称为`Raw`。

## 这些都是什么？
想要弄清Result和Raw的关系，我们需要捋清软件的分析流程。

一切都开始于采集。采集环节后，除了时间戳以外，我们会得到追踪点的坐标（中心点 或 鼻尖-中心点-尾根 或 色标）、身体长度、身体面积、像素点变化等基础信息。所有的指标和结果都是基于这些基础信息计算而来的。

### Result
在进入Result的计算之前，数据会先通过采集环节的轨迹平滑设置进行过滤。

而后根据用户选择的因变量，进行各种计算。
- 根据坐标计算移动距离、速度、航向、等
- 结合分析区设置中的分区信息，根据坐标计算被试动物所在区域
- 根据投影面积计算Mobility，根据像素点变化计算Activity
- 结合身体长度、投影面积、坐标等计算精细行为
- 其它更多的计算组合

这些指标，除了本身的计算方法以外，有的还需要或可以键入其它用户指定的参数，例如：
- 分析区的进入和退出阈值
- 移动 / 活动状态 / 多条件等参数需要用户指定的状态阈值
- 计算条件指定

这些数据还会通过数据选择配置中的要求，进行分类成组或截取，最终呈现给用户。

### Raw
Raw和Result最大的区别就是数据格式。Result是汇总的指标结果，而Raw是时序数据，即所有的结果不是汇总的，而是按照每个采样点一行排列的。

除了基础信息以外，Raw还会根据分析参数配置中选择的因变量，增加对应参数列。如果在数据选择配置中选择了分组，在列尾也会予以显示。

另外，Raw还会在文件的前40行上下输出Meta信息，包含实验的观察区设置名，实验控制设置名，检测设置名。

## 我们该用什么？
在EthoInsight立项之初，团队大部分人倾向使用Result作为输入进行分析。这样做的好处是显而易见的——**不重复计算**。这也是经理一开始提到的，*不去做EthoVision做过的事情。*
