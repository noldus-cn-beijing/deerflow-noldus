## 一、行为学同事侧能提供的材料

### 1. 各范式经典论文和代表综述

用于 CPT（继续预训练）阶段的知识注入。模型需要内化行为学领域的核心概念和研究范式。

**范围建议**：
- EPM / OFT / Zero Maze / LDB / TST / FST 各 3-5 篇经典论文（定义范式的那几篇）
- 焦虑行为、抑郁行为各 1-2 篇高引用综述
- 涵盖大鼠和小鼠两个物种

**格式**：PDF 即可，工程侧负责提取清洗。

### 2. EthoVision 各指标的计算方法文档

模型需要理解每个指标的含义、计算方式、单位、正常范围，才能正确解读分析结果。

**范围建议**：
- 核心指标定义：移动距离、速度、不动时间、在区域时间比例、进入次数等
- 衍生指标：thigmotaxis（OFT）等
- 每个指标的：语义含义、单位、典型值域、异常值特征
- 当前 `catalog/*.yaml` 里已有 `one_liner` + `direction_for_anxiety`，可作为骨架补充

**格式**：EthoVision XT 用户手册对应章节、或 Noldus 官方技术白皮书。

### 3. 行为学范式做法总结文档

教模型理解"用户描述的实验"对应什么分析流程。

**范围建议**：
- 每个范式的标准实验流程（动物准备、装置设置、时长、记录内容）
- Zone 划分约定（EPM 开放臂/封闭臂、OFT 中心/边缘等）
- 常见变体和注意事项（不同物种、不同光照条件、药物剂量范围）
- EV19 模板与学术范式的对应关系（行为学同事 PR 后补入）

### 4. Noldus 产品手册和白皮书

产品手册既有指标定义又有典型使用场景，是 CPT 的高质量语料。

**范围建议**：
- EthoVision XT User Guide（重点：数据导出格式、指标定义、zone 设置）
- Noldus 官方发表的 application notes / white papers

### 5. 真实 EthoVision 数据 + 人工分析报告配对

SFT 第二批数据的核心来源。几组"输入数据 → 期望输出"的配对样本，教模型"看到什么样的数据该出什么样的报告"。

**范围建议**：
- 每个 PRD 范式至少 1-2 组（EPM、OFT、Zero Maze、LDB、TST、FST）
- 每组包含：EthoVision 导出原始数据 + 你手写的完整分析报告
- 报告包含：实验设计描述、统计方法选择理由、结果解读、结论


### 6. Results 段落范例（中英双语）

微调设计里"报告生成"能力需要约 300 条 SFT 样本。已发表论文中的真实 Results 段落是最好的学习材料。

**范围建议**：
- 10-20 段来自已发表行为学论文的 Results 部分（脱敏处理）
- 涵盖不同统计方法（t 检验、ANOVA、非参数检验）
- 中英文各一半，中文部分体现学术写作习惯
- 标注每段使用的统计方法、报告的数据要素（均值、标准误、p 值、效应量）

### 7. 统计方法边界 case 说明

当前 `statistics-decision-tree.md` 覆盖了主干决策路径，但模型最容易翻车的是边界情况。

**范围建议**：
- 小样本（n=3）时 Shapiro-Wilk 不可靠，该怎么判断正态性
- 比例数据出现 0 或 1 时 logit 变换失效的替代方案
- 开放臂时间为 0（动物完全不进开放臂）怎么处理
- 多个指标同时检验时的多重比较校正策略选择（Bonferroni vs Holm vs BH-FDR）
- 效应量小但 p 显著 / 效应量大但 p 不显著时如何解读

## 二、开发同事侧需要提供/澄清的内容

### A. 训练目标对齐

| 问题 | 为什么重要 |
|------|-----------|
| CPT 先跑还是 SFT 先跑？ | 决定材料准备的优先级。CPT 只需文档语料，SFT 需要输入输出配对 |
| 微调后模型的角色定位？ | 做 agent（tool calling + 多步推理）vs 做分析助手（输入数据 → 输出报告），SFT 数据格式完全不同 |
| 训练分几个阶段，每阶段验收标准？ | 决定材料需要分成几批交付 |

### B. 基座模型确认

策略文档推荐 Qwen2.5-14B，设计文档用 Qwen3-8B，最新提议是 Qwen3-30B-A3B MoE + RTX 5090。**需要最终锁定**：

| 待确认项 | 影响 |
|----------|------|
| 具体模型名和参数量 | VRAM 需求、训练工具链、数据格式 |
| 量化方案（FP16 / Q4 / GPTQ） | 推理质量和速度的权衡 |
| 是否用 LoRA 还是全参数微调 | VRAM 需求差异 10x+ |

### C. 数据格式要求

| 问题 | 为什么重要 |
|------|-----------|
| 用什么训练框架？（LLaMA-Factory / Swift / Fireworks.ai / verl） | 数据格式不同（Alpaca / ShareGPT / ChatML / JSONL） |
| CPT 阶段接受什么格式？纯文本 / instruction-response / 混合 | 行为学同事提供的文档是否需要预处理 |
| SFT 数据的 system prompt 是否已固定？ | Agent 架构还在调整，过早固定 prompt 会浪费标注 |

### D. 训练基础设施

| 待确认项 | 用途 |
|----------|------|
| GPU 环境：几张卡、什么型号 | 训练时间估算 |
| 用 Fireworks.ai 托管还是本地训练 | 数据是否需要上传、隐私合规 |
| 推理评估环境（vLLM / Ollama / 其他） | 评估阶段的环境准备 |
| 训练预计什么时候启动？ | 材料准备的截止日期 |

### E. 评估标准

| 问题 | 为什么重要 |
|------|-----------|
| 怎么衡量"模型学会了行为学知识"？ | 是用 golden-case 做回归测试，还是另建评测集 |
| 可接受的最低准确率？ | 决定是否需要迭代训练 |
| 评估是自动化还是人工？ | 需要预留评估时间 |

## 三、材料优先级建议

按"对训练效果影响大 + 准备难度低"排序：

| 优先级 | 材料 | 用途 | 预计准备时间 |
|--------|------|------|-------------|
| P0 | 各范式经典论文 + 综述 | CPT 知识注入 | 1-2 天 |
| P0 | EthoVision 指标计算方法文档 | CPT + SFT | 1 天（已有手册） |
| P1 | 范式做法总结文档 | CPT + SFT | 3-5 天 |
| P1 | 统计方法边界 case 说明 | SFT 质量提升 | 2-3 天 |
| P2 | Noldus 产品手册/白皮书 | CPT 补充语料 | 1 天 |
| P2 | APA Results 范例（中英双语） | SFT 报告生成 | 2-3 天 |
| P3 | 真实数据 + 人工报告配对 | SFT 核心 | 持续积累（golden-case 体系） |
