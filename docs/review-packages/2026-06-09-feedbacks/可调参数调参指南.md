# 调参指南
issue #63 中提到的这些指标，在EthoVision的 分析参数配置 - 因变量 里成对出现。一个是连续值，一个是针对这个连续值施以1-2个阈值，切分成2-3类的状态值。常见的有 速度 - 移动（Velocity - Movement），Mobility - Mobility State，Activity - Activity State。

需要注意的是，这些指标几乎没有“推荐阈值”、“分组基线”或者“物种常模”这类判断依据。接下来我会描述实际在使用时的判断流程。由开发人员或AI模型把这个流程整理成算法或skill。

我们以常见的 activity - activity state 为例。
- 完成试验的同时采集好视频
- 在软件的分析参数设置中，先选三个activity的average interval（如1，3，5或1，5，10），再随便选一个activity state。
- 在集成可视化界面（上面是视频回放，中间是时间轴和播放控制按钮，下面是按轨道呈现的选择的因变量的数据可视化————连续值指标是折线图，状态值指标是分布图。视频和数据可视化可在时间轴使用滑块和播放控制按钮同步控制。），人工肉眼观察以下几点：
  1. 什么时候动物是不活跃的，什么时候是活跃的————不活跃的时候，activity的值大致是多少？
  2. 回到分析参数设置，调整不活跃的阈值，再回到集成可视化界面查看————不活跃的分布和视频是否一致？average interval的数值哪个能兼顾细节和降噪？
  3. 在步骤1和步骤2之间反复试错，直至找到一个合适的阈值。这个阈值需要不光适用于当前试次的被试动物，也需要适配于试验内的所有动物————所有被试需要在同一标准下进行衡量才有可比性。


## 跨范式共享
需要注意的两点：
1. **悬尾和强迫游泳不需要测量速度（velocity）**。
2. **不要想物种典型值**。典型值/基线/常模需要大量的，稳定的，有共同特点和生活方式的群体才能获得。不同品系/年龄的鼠，在不同的实验室，不同的饲养环境，不同的造模/给药/处理方式，会造成差异，甚至截然相反的结果。

### velocity_threshold
有了上面的内容后，其实这里最需要关注的只有一种情况——鼠窝在一个地方不动。这种情况下，会有大段大段的速度几乎为0的情况。（为什么说是几乎？因为哪怕它窝着不动，在不开最小移动距离滤镜的情况下追踪点在各帧之间也会有坐标的抖动，带来虚空移动距离）所以这里的速度阈值可以定的很低，比如0.1cm/s。
这一点在任何轨迹追踪范式中都通用。这也意味着，**原地行为为主的实验（强迫游泳FST和悬尾TST）不需要测量这一指标。**
1. 生理基础: 判断移动或不动的阈值。
2. 调高会怎样 / 调低会怎样:调高会有更多的帧判定为不动，immobility膨胀，可能误判为高焦虑；调低问题不大。
3. 物种典型值:不要再想这个了。
4. 何时该调 / 何时不该调:不需要动。因为移动这个指标和activity不一样，不动就是不动，只有追踪点抖动带来的虚空移动速度和距离。

### velocity_min_duration
同样不适用于原地行为为主的实验。
最少持续2-3秒就算不动。只要一个具体数值的话我给2秒。
4点影响同velocity_threshold

## EPM
### motor_low_entries_threshold
我倾向设置为1-2。
1. 生理基础:这基本代表着鼠压根没有进开放臂/鼠就朝着开放臂探了下头。
2. 调高会怎样 / 调低会怎样:高了的话，进开放臂好多次的鼠会被误判高焦虑；调低只要不是0就行（1和2已经没有什么调低空间了）
3. 物种典型值:不要再想这个了
4. 何时该调 / 何时不该调:基本不用调。

## LDB
### signal_low_transition_threshold
设置为2次。
1. 生理基础：这基本代表着鼠直接进了黑箱不出来了
2. 调高会怎样 / 调低会怎样：高了的话常来亮区的鼠会被误判高焦虑；调低只要不是0就行（1和2已经没有什么调低空间了）
