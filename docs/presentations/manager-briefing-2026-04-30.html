🔧 关键修复：Event Loop 崩溃问题已解决

❌ 问题（4月28日发现）

用户上传数据让 Agent 分析时，后台出现 "Event loop is closed" 错误导致崩溃。

通俗解释：Agent 在"记忆更新"和"主对话"两个任务之间共用了一条"网络连接通道"。记忆更新完成后关闭了这条通道，主对话再想用时就找不到了——类似于两个同事共用一台电脑，一个人用完关机了，另一个人回来发现电脑打不开了。

✅ 修复方案（4月29日完成）

采用上游 DeerFlow 框架的最终方案：让"记忆更新"和"主对话"走完全独立的两条通道，互不干扰。

通俗解释：给两个同事各配一台电脑，各用各的，再也不会互相影响。

🤖 当前 Agent 能力一览

EthoInsight 是一个面向行为学研究员的 AI 助手。用户上传 EthoVision XT 实验数据后，Agent 自动完成统计分析、专业解读和报告生成。

📊 自动数据分析

★★★★☆

上传实验数据 → 自动解析轨迹 → 计算行为指标 → 统计检验 → 生成图表 → 输出报告。shoaling 范式已通过端到端验证。

🧠 智能统计决策

★★★★★

自动检测数据是否符合正态分布 → 选择参数或非参数检验方法 → 输出 APA 格式统计结果。研究员无需手动选择统计方法。

🔍 数据洞察 & 异常检测

★★★☆☆

能按受试者逐一排查异常值、识别混杂因素。例如：检测到某只动物的游泳距离异常（70 秒），自动排除后重新分析。

📝 结构化报告生成

★★★★☆

自动生成包含方法、统计结果、图表、结论的完整分析报告。支持追问和深入解读。

🗣️ 多轮对话 & 追问

★★★★☆

分析完成后可以追问："为什么这个 p 值不显著？""能换个统计方法吗？"Agent 能结合数据上下文回答。

🛡️ 质量关卡机制

★★★★☆

内置 3 道质量关卡：① 范式确认 → ② 数据质量检查 → ③ 统计方法审核。防止"跑偏"和错误分析。

📚 知识问答

★★☆☆☆

能回答行为学方法学问题、实验设计建议。目前知识面较窄（noldus-kb 论文库暂不可用），依赖内置 skill 知识。

🔄 训练数据飞轮

★★★☆☆

每次 Agent 对话自动录制为训练数据。后续用于微调自有模型，让 Agent 越来越"懂"行为学。
