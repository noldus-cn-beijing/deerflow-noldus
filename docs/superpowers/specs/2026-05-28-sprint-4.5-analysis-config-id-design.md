### Sprint 4.5 要解决的问题

用户改了参数（如 "immobility velocity_threshold 从 30 改到 5"）后，下面两件事必须可追溯：

1. **可重放**：同 input + 同 overrides 必然产出同 `analysis_config_id`；不同 overrides 必然不同 id
2. **可对比**：用户问"上次的报告和今天的报告差在哪"，agent 能从两个 id 反查参数差异

### 设计原则（grill 锁定）

**B deterministic hash（grill Q6 锁定）**：
```python
analysis_config_id = sha256(
    canonical_json(catalog_default_snapshot + overrides)
)[:16]
```
- 输入：catalog 当前所有 paradigm 的 default 参数快照 + 用户的 overrides dict
- canonical_json：sorted keys、no whitespace、UTF-8
- 取前 16 字符（够稀疏，64-bit 碰撞空间）
- **关键约束**：catalog YAML 变化会自动改变 default snapshot 部分 hash —— "升级了 catalog 但忘改 overrides" 这种情况会产生新 id，可被检测（Sprint 5.5 lineage 封印的天然衔接）

**iii overrides 位置（grill Q5 锁定）**：
```json
// experiment-context.json
{
  "paradigm": "forced_swim",
  "paradigm_cn": "强迫游泳测试",
  "category": "anxiety_depression",
  "subject": "rodent",
  "ev19_template": "FST-Standard",
  "gate_completed": ["gate1_paradigm", "gate2_quality_acknowledged"],

  // === Sprint 4.5 新增 ===
  "parameter_overrides": {
    "velocity_threshold": 5.0,
    "velocity_min_duration": 25
  },
  "analysis_config_id": "a3b7c1d2e4f8a0b6"
}
```

`prep_metric_plan_tool` 自动从 context 读 `parameter_overrides`，写到 `/mnt/user-data/workspace/overrides_<config_id>.json`，调 `catalog.resolve --overrides-file <path>`。lead **不需要**手动管 override 文件路径。

**b 展示位置（grill 锁定）**：
- 报告末尾："本次分析标识：`a3b7c1d2e4f8a0b6`"
- Sprint 7 假设面板顶部（Sprint 7 自行实现，本 sprint 不做 UI）

### 2.1 新增 `compute_analysis_config_id()` helper

**位置**：`packages/agent/backend/packages/harness/deerflow/agents/middlewares/experiment_context.py`

在文件顶部 helper 区追加：

```python
def compute_analysis_config_id(
    catalog_default_snapshot: dict[str, Any],
    overrides: dict[str, float | int | str] | None = None,
) -> str:
    """Deterministically compute a 16-char hex id from catalog defaults + overrides.

    Sprint 4.5 §1 grill-locked design:
      analysis_config_id = sha256(canonical_json(defaults + overrides))[:16]

    Args:
        catalog_default_snapshot: Full default-value snapshot from the active
            catalog, e.g. {"forced_swim.velocity_threshold": 30.0, ...}.
            Use a flat dotted-key dict so YAML re-organization doesn't
            silently change the hash.
        overrides: User-provided parameter overrides. None and {} produce
            identical ids (canonical "no override" state).

    Returns:
        16-char lowercase hex digest of the sha256 over canonical JSON.
    """
    import hashlib
    import json as _json

    payload = {
        "defaults": dict(sorted(catalog_default_snapshot.items())),
        "overrides": dict(sorted((overrides or {}).items())),
    }
    canonical = _json.dumps(payload, ensure_ascii=False, sort_keys=True, separators=(",", ":"))
    return hashlib.sha256(canonical.encode("utf-8")).hexdigest()[:16]


def get_catalog_default_snapshot() -> dict[str, float | int | str]:
    """Read the active catalog and return a flat dotted-key dict of all
    `tunable_by_user: true` parameter defaults across all paradigms.

    Format: {"<paradigm>.<param_name>": <default_value>, ...}
    """
    from ethoinsight.catalog.loader import iter_tunable_parameters

    return {
        f"{paradigm}.{param}": default
        for paradigm, param, default in iter_tunable_parameters()
    }
```

`iter_tunable_parameters()` 是 ethoinsight 侧已知 catalog loader（Sprint 2a 引入）需要导出的迭代器；若 Sprint 2a 没暴露这个 helper，Sprint 4.5 agent 顺手在 `catalog/loader.py` 加一个（一并 commit）。

### 2.2 改动 `set_experiment_paradigm` tool

**位置**：`packages/agent/backend/packages/harness/deerflow/agents/middlewares/experiment_context.py:174`

新增 `parameter_overrides` 可选参数 + 在 Gate 1 / Gate 2 / 单独改 overrides 三种 path 下都重算 `analysis_config_id`：

```python
@tool("set_experiment_paradigm", parse_docstring=True)
def set_experiment_paradigm_tool(
    paradigm: str | None = None,
    paradigm_cn: str | None = None,
    category: str | None = None,
    subject: str | None = None,
    ev19_template: str | None = None,
    acknowledge_quality: bool = False,
    # === Sprint 4.5 新增 ===
    parameter_overrides: dict[str, float | int | str] | None = None,
    workspace_dir: str = "/mnt/user-data/workspace/",
    runtime: ToolRuntime[ContextT, ThreadState] = None,
) -> str:
    """Record paradigm choice / quality acknowledgement / parameter overrides.

    Three usage modes (Sprint 4.5 adds the third):
      1) Gate 1 paradigm confirmation: pass paradigm/paradigm_cn/...
      2) Gate 2 quality acknowledgement: pass acknowledge_quality=True
      3) Parameter override update: pass parameter_overrides={"velocity_threshold": 5.0, ...}
         → re-computes analysis_config_id; existing gates preserved

    Args:
        ...
        parameter_overrides: When provided, merges into existing overrides dict
            (None values explicitly remove a key). Triggers a fresh
            analysis_config_id recomputation regardless of which gates ran.

    Returns:
        JSON containing analysis_config_id (always present after Sprint 4.5).
    """
```

**关键行为变更**：每次写 experiment-context.json 都必须**重新 compute** `analysis_config_id`，即使是 Gate 1 / Gate 2 操作：

- Gate 1 路径：catalog_default_snapshot 已确定，overrides 默认 `{}` → 仍生成一个稳定 id（用户没改参数时也有 id）
- Gate 2 路径：overrides 不变 → id 不变（重算只是 idempotent 验证）
- Override 更新路径：merge 进现有 overrides → 算新 id

**实现要点**：

```python
# 在 set_experiment_paradigm_tool 函数体内，写文件前统一做：
existing = read_context(actual_workspace) or {}

# 合并 overrides（None value = 删 key）
current_overrides = dict(existing.get("parameter_overrides", {}))
if parameter_overrides is not None:
    for k, v in parameter_overrides.items():
        if v is None:
            current_overrides.pop(k, None)
        else:
            current_overrides[k] = v

# 重算 analysis_config_id
defaults = get_catalog_default_snapshot()
new_id = compute_analysis_config_id(defaults, current_overrides)

# 写入
existing["parameter_overrides"] = current_overrides
existing["analysis_config_id"] = new_id
# ... 其他 Gate 字段照旧 ...
write_context(actual_workspace, existing)
```

### 2.3 改动 `_read_analysis_config_id()`

**位置**：`packages/agent/backend/packages/harness/deerflow/tools/builtins/seal_handoff_tools.py:57-71`

把占位逻辑改为真读：

```python
def _read_analysis_config_id(workspace: Path) -> str:
    """Read analysis_config_id from experiment-context.json. Returns "PENDING"
    only when the context file is missing or malformed (defensive).
    """
    path = workspace / "experiment-context.json"
    try:
        if not path.exists():
            return "PENDING"
        ctx = json.loads(path.read_text(encoding="utf-8"))
        config_id = ctx.get("analysis_config_id")
        if isinstance(config_id, str) and config_id:
            return config_id
        return "PENDING"
    except (OSError, json.JSONDecodeError) as e:
        logger.warning("Failed to read analysis_config_id from %s: %s", path, e)
        return "PENDING"
```

**注意**：把 `"PENDING_SPRINT_4.5"` 改为 `"PENDING"` — Sprint 4.5 完工后，看到 `PENDING` 才是真异常（context 文件读不出），不是"还没实施"。Sprint 0 的占位字符串清掉。

## 4. 风险与缓解

| 风险 | 缓解 |
|---|---|
| `get_catalog_default_snapshot()` 依赖 ethoinsight catalog API（Sprint 2a 引入），若 API 变动 hash 输入会变 | T1 用 dotted-key flat dict 而不是原 YAML 嵌套结构；catalog YAML 重组不会改 hash 输入；只有 default 数值变化才会改 |
| 用户改 catalog YAML 但忘了对应 paradigm md（Sprint 4） → catalog 漂移但调参指南不变 | hash 输入只看 catalog default snapshot；这是**特性**而非 bug——同 overrides 但 catalog 升级会产新 id，提示"环境变了" |
| Gate 1 时计算 id 但 ethoinsight catalog 还没 import 完 | T1 用 lazy import；catalog API 不可用时 fallback `compute_analysis_config_id({}, overrides)`（仅基于 overrides，记日志）|
| 老 thread（无 experiment-context.json）走到 seal 流程报错 | `_read_analysis_config_id` 已防御 `path.exists()` 不存在返回 `"PENDING"` |
| `parameter_overrides=None` 与 `parameter_overrides={}` 语义不同导致 LLM 困惑 | tool docstring 明确："不传 = 不动；传 {} = 清空所有 overrides；传 {'x': None} = 仅删 x 这一个 key" |
| analysis_config_id 在 manifest 与 handoff 不一致 | seal_handoff_tools.py 同一函数内先读 id 再注入 + 写 manifest，单一来源 |
| Sprint 5.5 lineage 封印对 PENDING 状态的处理未定 | T5 改成 `"PENDING"` 而非 `"PENDING_SPRINT_4.5"`，给 Sprint 5.5 一个干净的"异常态"信号 |
