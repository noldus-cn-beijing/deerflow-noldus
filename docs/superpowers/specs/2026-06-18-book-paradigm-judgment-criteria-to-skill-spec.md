# Spec：把行为学书的判读增量补进 6 范式 by-experiment skill —— 充实 data-analyst 的混杂排查判据

## 〇、给实施 agent 的一句话

行为学同事把《动物行为实验指南》(680 页) 切成 `~/behavioral-book/by-paradigm/` 范式章节。经「书原文 vs 已合 skill」逐范式差距分析，发现 **6 个 v0.1 范式（EPM/Zero Maze/OFT/FST/TST/LDB）的 by-experiment skill 已守住核心防线（判读方向 + 绝对阈值禁令 + 主要混杂），但 data-analyst 排查混杂时少了若干具体检查项**。本 spec 把这些**通过四重严格筛选**的增量，按项目铁律改写成组间口径，补进对应 skill 的 `## 脱险点 / 数据质量风险` 和 `### 解读方向与口径` 段。**这是判据充实，不是改判读哲学**——所有增量都不引入绝对阈值/绝对程度词、不含厂商名、EV19 轨迹能算。

> **与另一份 spec 的关系（必读）**：本 spec 与 `2026-06-18-data-analyst-thinking-overload-spec.md` **互补、不冲突**。后者砍掉 data-analyst 的机械工作量（参数审计 step 2.8、逐条搬运 outlier）并强化「必 read by-experiment/<paradigm>.md」；本 spec 正是**充实那份被 read 的判据**，填补「砍掉参数审计后 data-analyst 判读靠什么」的空缺。**本 spec 的增量一律进 by-experiment skill（叙述层判据），绝不进 step 2.8 参数审计决策树**（那段已被另一 spec 删除）——任何把「品系核对/钟摆参数核对」写回 step 2.8 的做法都是错的。

## 二、设计原则

### 2.1 组间口径改写（铁律第 9 条）

书里凡用绝对数值/程度词的（如「总运动<1000cm 是运动抑制」「不动>Xs 即抑郁」），入 skill 前**必须改写成组间比较 + 效应量口径**：「与对照组相比总入臂显著下降（伴开臂下降）→ 判运动抑制混杂而非焦虑」。绝不写死阈值。

### 2.2 量化基线预期 ≠ 绝对阈值（重要区分）

「品系基线不动差异可达 10 倍」「C57 对照常 >100s」这类是**离群判读的预期参照**（帮 data-analyst 判断「这个悬殊是品系正常差异还是异常」），**不是判读阈值**。入 skill 时措辞为「跨品系基线差异可达数量级，看到对照组间基线悬殊应先归因品系而非异常个体」，**不写成「>100s 即抑郁」**。

### 2.3 增量是「检查项 + 判读动作」，不是知识堆砌

每条增量落成 data-analyst 能执行的形式：**何时触发（看到什么数据特征）→ 查什么 → 怎么判**。避免写成教科书段落（防 skill 膨胀诱发 thinking 过载，呼应另一 spec 的根因）。

### 2.4 插入位置固定

- **混杂/离群/质量类**增量 → `## 脱险点 / 数据质量风险` 段追加条目。
- **解读方向/指标取舍类**增量 → `### 解读方向与口径` 段追加。
- 不新建顶层段落（保持 6 范式结构一致：范式定位/模型概述/设计原理/指标与判读/脱险点/与其他范式区分）。

### 2.5 不碰 reward-criteria YAML、不改 catalog

reward YAML 是给 RL 的规则层（双消费物理分离）。本 spec 只动 by-experiment skill（叙述层）。指标公式仍以 catalog 为 SSOT，skill 不重复。

## 三、改动清单（按范式 + 增量逐条，含书出处与改写措辞）

> 每条给：【类别】【书出处】【改写后入 skill 的措辞（组间口径）】【插入段】【铁律守护点】。实施 agent 按此补写，措辞可润色但**不得引入绝对阈值/程度词/厂商名**。

### 3.1 EPM（`epm.md`）

**E1【混杂排查】预暴露混杂**（书 §6.1）：测试前暴露新环境会**增加**开臂活动与探索。
→ 入「脱险点」：「若各组测试前的环境处理不一致（如仅处理组做了新环境预暴露），开臂指标升高可能源于预暴露而非干预效应——判读前确认各组前处理一致。」铁律：纯设计混杂，无阈值。

**E2【离群判读】掉落/冻结的剔除标准**（书 §9.1/9.2）：<1% 动物从开臂掉落，数据应剔除；噪音致冻结的动物应剔除并透明报告剔除数/总数。
→ 入「脱险点」：「轨迹出现断层/异常静止时，区分掉落（开臂边缘后轨迹消失）、外部噪音致冻结（应剔除并在报告注明剔除 n/总 n）、与真实低探索个体；剔除决定要可追溯。」铁律：透明度要求，无阈值。

**E3【研究建议】重复测试可行条件**（书 §6.2）：间隔≥3 周 + 换房间可重测而开臂基线不下降。
→ 入「脱险点」（已有耐受条目，扩写）：「若研究者坚持重复测试 EPM，提示其满足『间隔≥3 周且更换房间』才能避免一次试验耐受；否则二次测试开臂下降可能是耐受而非干预效应。需重复测量优先建议 Zero Maze。」

### 3.2 Zero Maze（`zero_maze.md`）

**Z1【解读方向】犹疑次数 × 开放区的多指标权衡**（书 §4 指标）：犹疑反映接近-回避**冲突瞬间**，与最终选择（开放区时间）互补。
→ 入「解读方向与口径」：「同时看开放区时间与犹疑次数：开放区↓且犹疑↑（组间）提示冲突高但最终回避；开放区↓且犹疑↓提示回避而冲突弱——二者机制不同，不要只看单指标下结论。」铁律：组间口径，无阈值。

**Z2【指标取舍澄清】stretch-attend 不可算**（书 §4 高评价 stretch-attend）：
→ 入「解读方向与口径」一句澄清：「书中高度评价的伸展注意（stretch-attend）因依赖身体姿态、EV19 轨迹无法自动推导，本系统不产出；用犹疑次数 / head-dipping（需用户有探头分析区列）作过程型替代。」铁律：守 D，不编造。

### 3.3 OFT（`open_field.md`）

**O1【混杂排查】运动抑制 vs 焦虑的三步分离判读**（书 §脱险点）：
→ 入「脱险点」（升级现有一句）：「区分运动抑制与焦虑用三步组间比较：① 总移动距离（vs 对照显著↓→疑运动抑制）② 中心区比例（vs 对照↓→疑焦虑样分布改变）③ 是否伴刻板/聚集（疑抑郁）。三者结合判断，不要单看任一指标的绝对值。」铁律：明确「组间比较，不写绝对距离阈值」。

**O2【数据质量】中心区定义一致性把关**（书 §仪器/脱险）：实验室间中心/外周划分不统一→结果不可比。
→ 入「脱险点」：「拿到新数据先确认 zone 定义（中心/边缘如何划），而非直接判焦虑水平；两份数据 zone 定义不同则不可混合比较——与列语义对齐 HITL 衔接。」铁律：方法学卡点，无阈值。

### 3.4 FST（`forced_swim.md`）

**F1【混杂排查】兴奋剂假阳性 + 运动测试顺序**（书 4.2 §补充测试）：减少不动可能是运动兴奋而非抗抑郁；动物复用时**运动测试须在 FST 之前**（FST 有长期行为影响）。
→ 入「脱险点」（扩写现有假阳性条目）：「不动下降（尤其疑似兴奋剂样作用）须配独立运动活动测试排除假阳性；若同一动物复用，运动测试应在 FST 之前，否则 FST 的长期影响污染运动基线。报告中无独立运动测试时，不动下降不可单独判为抗抑郁。」铁律：流程建议，无阈值。

**F2【离群判读】品系基线量化预期**（书 4.2）：小鼠品系基线不动差异**可达 10 倍**；Wistar-Kyoto > SD（伴攀爬↓）。
→ 入「脱险点」：「跨品系数据基线不动可差数量级——看到对照组间基线悬殊，先归因品系差异（预期内），而非误判为异常个体或实验故障；品系混杂时组间比较本身需先按品系分层。」铁律：§2.2 量化预期非阈值。

**F3【指标取舍】SSRI × FST 版本适配**（书 4.1/4.2）：原始 FST 对 SSRI 检测不可靠；修改版（攀爬/游泳分解）才能区分。
→ 入「解读方向与口径」：「按 FST 版本判读：若干预为 5-HT 能（SSRI 类）但数据来自原始 FST（不分攀爬/游泳），提示其检测可靠性有限，报告应注明；修改版才能区分神经递质机制。」铁律：方法学适配，无阈值。

**F4【解读方向】攀爬/游泳 → 机制 + 反证检验**（书 4.2）：儿茶酚胺能↑攀爬、5-HT 能↑游泳。
→ 入「解读方向与口径」（扩写）：「修改版 FST：仅攀爬↑（组间）提示儿茶酚胺能、仅游泳↑提示 5-HT 能、二者皆↑提示多靶点。若数据模式与声称机制矛盾（如只增游泳却称儿茶酚胺能作用），标红待复查。」铁律：药理映射，组间口径。

**F5【混杂排查】季节性混杂**（书 4.1）：冬季不动↑、夏季↓。
→ 入「脱险点」：「同实验室跨批/长期数据汇总时，注意季节可能混杂基线不动（冬↑夏↓）；跨季节对比应分季节看效应，避免季节主效应掩盖干预效应。」铁律：混杂识别，无阈值。

### 3.5 TST（`tail_suspension.md`）

**T1【混杂排查】爬尾混杂（C57BL/6 特有）**（书 5.2）：该品系常爬自己尾巴，违反不可逃避假设，应剔除；防爬管已证不影响药效。
→ 入「脱险点」：「C57BL/6 数据：若未用防爬管且视频可见爬尾，该样本应排除（爬尾违反 TST 不可逃避前提）；排除后需重算 n 与统计功效。」铁律：品系特异混杂，无阈值。

**T2【混杂排查】体重/尾巴承重 → 范式适用性**（书 5.1/5.2）：TST 对较重动物（大鼠、肥胖模型）有尾痛混杂，应改 FST 或补独立运动测试。
→ 入「脱险点」：「实验对象为大鼠/肥胖模型/体重偏大小鼠时，TST 的尾部承重可能因疼痛被误读为不动——提示改用 FST 或补独立运动活动测试排除运动能力混杂。」铁律：适用性评估，无阈值。

**T3【指标取舍】Activity（TST）vs Mobility（FST）不可混用**（书 5.1/5.2 + skill 已有一句）：
→ 入「解读方向与口径」（强化）：「TST 用 Activity/Activity State、FST 用 Mobility/Mobility State，二者不可混用；若发现指标参数被混（如用 Mobility 判 TST 不动），导致绝望判读错误，应叫停核对。」铁律：技术差异，无阈值。

**T4【解读方向】TST 全程评分 vs FST 后段截断**（书 5.2）：TST 小鼠早期即倾向不动，**全 6 分钟评分**，不能套 FST「跳过前段只算后段」。
→ 入「解读方向与口径」：「TST 早期不动本身有信息，按全程评分判读；不要套用 FST 的『只取后几分钟』窗口，否则丢失早期不动信号。」铁律：方法学，无阈值。

### 3.6 LDB（`light_dark_box.md`）

**L1【解读方向】潜伏期的起始位置二重性**（书 §指标）：潜伏期含义随起始位置反转。
→ 入「解读方向与口径」：「潜伏期判读须对应起始位置：起始=明箱时，进暗箱潜伏期反映避明强度、返明箱潜伏期反映回避恢复；起始=暗箱时，进明箱潜伏期反映探索动力。报告潜伏期须注明起始位置。」铁律：参数定义澄清，无阈值。

**L2【数据质量】穿梭次数双重性**（书 §指标，与 OFT 类似但机制不同）：
→ 入「解读方向与口径」（强化现有）：「穿梭次数既是焦虑指标又是运动量指标：仅当穿梭↑『不伴随自发运动↑』（组间）才提示抗焦虑；若一般活动同时↑，穿梭↑可能是运动假阳性。」铁律：组间口径，无阈值。

### 3.7 通用澄清句（OFT + Zero Maze 各补一句）

OFT/Zero Maze 在「解读方向与口径」补一句（若已有近似则跳过）：「书中讨论的直立行为细分（无支撑/支撑 rearing）因 EV19 精细行为分类器信效度不佳、本系统不产出，报告不得编造其数值；相关探索倾向用中心区/开放区指标体现。」

## 四、验证（本 spec 改的是 skill md，非代码，验证以「内容正确性 + 不破坏契约」为主）

### 4.1 铁律静态扫描（必做）

对 6 个改后 skill 跑禁词扫描，断言**新增内容不含**绝对阈值/绝对程度词模式：grep 类似 `正常范围`、`即(高|低)(焦虑|抑郁)`、`<\d+%.*焦虑`、`>\d+s.*抑郁`、厂商名/具体药物剂量（如 `mg/kg`）。对照 `output-constitution.md` 禁词清单。任一命中 → 该条改写不合格，回炉。

### 4.2 增量可追溯（每条注出处）

每条新增内容在 skill 里**不必**写书页码（保持叙述干净），但实施 agent 须在 PR 描述里列「增量条目 ↔ 书出处 ↔ 改写前后」对照表，供 review 核验未夹带私货。

### 4.3 EV19 可算性复核

新增涉及的每个指标/行为，确认 catalog 有对应登记或 EV19 轨迹能产出；算不出的（stretch-attend/rearing）只能以「澄清不支持」形式出现，不得作为「建议补测的判读指标」。

### 4.4 结构不漂移

6 个 skill 顶层段落仍为固定 6 段（范式定位/模型概述/设计原理/指标与判读/脱险点/与其他范式区分），未新建顶层段；增量只进「脱险点」或「解读方向与口径」。

### 4.5 skill 体积守护（呼应 thinking 过载根因）

每个 skill 增补后总体积控制（建议单文件 ≤ ~9KB，与现有 pendulum-params 量级相当）；增量是**检查项式短条目**，不是长教科书段落——防 skill 过长反而加重 data-analyst 阅读/thinking 负担。

### 4.6 不需跑后端测试

本 spec 不改 Python。但若实施 agent 顺手动了 catalog/代码（不应），则触发对应全量回归——正常情况下只动 6 个 md，无需 `make test`。

## 六、风险与注意事项

1. **绝不把增量写回 step 2.8 参数审计**（§〇）：FST/TST 的「品系/钟摆参数核对」诱人写成参数审计项，但 step 2.8 已被另一 spec 删除——这些一律作为 by-experiment 判据，data-analyst read 后用于判读，不进决策树。
2. **量化基线预期 vs 绝对阈值的红线**（§2.2）：「品系差异 10 倍」「C57 常 >100s」只能作离群归因参照，**严禁**写成「>100s 即抑郁」式判读阈值。这是最容易踩铁律的点，review 重点查。
3. **书的绝对阈值表述是陷阱**：书里「预期结果」段大量「焦虑↓→开臂↑」反链式绝对表述，照搬即违宪。每条增量都要主动改写。
4. **EV19 不支持的别当增量**：rearing/stretch-attend 判读价值高但算不出，写成「建议补测」会诱导 data-analyst 索取不存在的指标 → 前端幻影/报告编造。只能澄清不支持。
5. **与 thinking-overload spec 的实施顺序**：两 spec 独立，可并行实施（改不同文件：本 spec 改 6 个 md，另一 spec 改 data_analyst.py/run_metric_plan_tool.py/handoff_schemas.py）。无冲突。但 review 时确认本 spec 强化的「判读判据」与另一 spec 强化的「必 read by-experiment」措辞对齐（data-analyst read 的就是本 spec 充实后的文件）。
6. **范式书还有 119 个范式未提炼**（README「下一步」）：本 spec 只做 v0.1 的 6 个；其余范式等 v0.1 后或同事按需 PR，不在本 spec 范围。
