### 3.2 核心数据流

```
用户上传 EthoVision 文件
  → ExperimentContextMiddleware 自动提取元数据（范式、subject、时长）
  → Lead Agent 向用户确认分组
  → task(code-executor): 读模板 → 修改参数 → 执行 → 写 handoff
  → task(data-analyst): 读 output → 解读结果 → 查 noldus-kb → 写分析报告
  → task(report-writer): 读数据+分析 → 写 APA 格式报告
  → Lead Agent 整合返回用户
```

## 4. 关键决策记录

| 决策 | 选择 | 原因 |
|------|------|------|
| 知识库接入方式 | noldus-kb MCP（非 CLI、非 YAML） | noldus-kb 在远端服务器，MCP 解耦最干净；noldus-kb 开发阶段变动不影响 EthoInsight |
| 默认 subagent | 只注册 code-executor/data-analyst/report-writer | general-purpose 和 bash 是 DeerFlow 通用的，EthoInsight 不需要 |
| Skills 配置 | 7 启用 / 11 禁用 | 保留数据分析、图表、图片生成、PPT、研究、skill 发现；禁用视频、播客、商业分析、开发工具等 |
| image_search tool | 移除 | 行为学分析无图片搜索需求 |
| Memory 系统 | 不修改 | DeerFlow 已做好，通用 fact 提取足够 |
| Group polarity | atan2(dy,dx) + 5帧平滑 | Shoaling 无 heading 列，从坐标差推导 |
| 统计检验选择 | 自动（Shapiro-Wilk → t-test/Mann-Whitney） | 用户不需要手动选择检验方法 |
