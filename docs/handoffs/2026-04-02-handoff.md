## 1. 当前任务目标

为 Noldus 公司设计并开发 **EthoInsight** — 一个基于 AI Agent 的动物行为学端到端数据分析产品。用户上传 EthoVision 导出的 raw time-series 数据 + 输入实验上下文，Agent 自动完成分析并生成可用于论文的报告。

**产品定位：** 不是可视化仪表盘，是"替实验人员做分析、给结论"的 AI 助手。目标用户是研究生、博后、技术员——知道怎么做实验但不擅长数据分析的人。他们的最终目的是发论文，通常需要 3 个左右行为范式互相印证一个结论。

### EthoVision 数据格式（实测）

- 编码：UTF-16 LE with BOM
- 分隔符：分号 `;`
- Header：37-39 行 metadata（实验名、试验名、对象名、检测设置、开始时间、持续时间等）
- 列：试用时间、录制时间、X中心、Y中心、区域、面积变化、伸长、移动距离、Velocity、Result 1
- 单位：秒、毫米、毫米²、毫米/秒
- 缺失值：`"-"`（字符串 dash）
- 采样率：~30Hz（0.033s 间隔）
- 每个文件约 9000 行数据（5 分钟 trial）

### noldus-kb 服务

- 6200+ 论文和手册的向量检索服务
- 9 个 Noldus 产品、15 个实验范式、15 个行为学术语
- 三种访问方式：CLI / MCP / REST API
- Agent 通过沙箱 bash 调用 `noldus-kb --format json search/paradigm/term/product`
- 服务端口：API 7000, MCP 7001（用户已修改默认端口）
- CLI 默认 API URL：`http://localhost:7000`（用户已修改）

### DeerFlow 架构要点

- **lead_agent** 通过 `make_lead_agent(config)` 创建，`apply_prompt_template()` 动态生成 prompt
- **中间件链**：12 个中间件硬编码有序列表（ThreadData → Uploads → Sandbox → ... → Clarification）
- **ThreadState**：通用字段（sandbox, artifacts, todos, uploaded_files, viewed_images）
- **工具注册**：config.yaml `tools` section，`module: package.module:function` 格式
- **记忆**：LLM 提取 facts 存 memory.json，注入时取 top 15 facts + context（最多 2000 tokens）
- **子 Agent**：内置 general-purpose 和 bash，最多 3 并发，15 分钟超时
- **文件上传**：`POST /api/threads/{thread_id}/uploads`，用 markitdown 转换
- **沙箱**：三种模式（local/Docker/K8s），虚拟路径 `/mnt/user-data/{workspace,uploads,outputs}`

## 4. 关键发现和决策

### 行为学同事的反馈（重要）

行为学没有绝对的"常模"或"正常范围"。不存在"某品系小鼠开放臂停留时间低于 X% 就是焦虑"这种绝对阈值。核心方法论是**组间对比**——设置对照组，和不同剂量/处理的实验组做相对比较。

**影响：** 所有文档已修正。知识库存储的不是"阈值"，而是：每种范式该计算哪些指标、每个指标的生物学含义、组间比较时该选什么统计方法、需要检查的混杂因素、效应量的解读参考。

### 用户的真实使用场景

发论文是最终目的。一篇论文通常需要 ~3 个行为范式互相印证一个结论（如"吃了我的药，焦虑下降了"），还要结合生理学、遗传测序、解剖等其他证据。单个实验（如水迷宫）可能持续 1-2 周。有的学生当天分析，有的攒到最后。

**影响：** 产品定位从"分析单个实验"变为"帮用户把多个实验的行为学证据链串起来，为论文提供行为学部分的完整支撑"。Session 记忆的价值在于跨实验的上下文积累。

### Fork vs 配置扩展

配置模式能做：注册工具、接入 LLM、注入 Skills prompt、noldus-kb 调用。
配置模式做不到：替换 System Prompt 基础角色、扩展中间件链、扩展 ThreadState、处理 EthoVision 上传格式、特化记忆提取 prompt、注册领域子 Agent、定制前端。
结论：Fork。
