### 3.2 DeerFlow 关键架构（已确认）

- **Agent 框架**在 `backend/packages/harness/deerflow/`，不是 `backend/app/`
- **harness 包**可发布，**app 层**不可发布。app 可以 import deerflow，反之不行
- **Subagent 注册**：`deerflow/subagents/builtins/__init__.py` → `BUILTIN_SUBAGENTS` 字典
- **Subagent 配置**：`SubagentConfig(name, description, system_prompt, tools, disallowed_tools, model, max_turns, timeout_seconds)`
- **Lead Agent 通过 task() 工具委派**：参数 `description`, `prompt`, `subagent_type`
- **subagent 上下文隔离**：独立 agent 实例，不继承主对话，但共享 sandbox 文件系统
- **工具按 group 分组**，subagent 通过 tools allowlist 指定可用工具
- **prompt 中 subagent 列表目前是硬编码的**（只有 general-purpose 和 bash），需改为动态生成
- **DeerFlow 有 `ask_clarification` 工具**：lead agent 可以用它中断执行、向用户提问

### 3.3 EthoVision 数据格式（已确认）

- 编码：UTF-16 LE + BOM (FF FE)
- 分隔符：分号 (`;`)
- 换行：CRLF
- Header：第 1 行声明 header 行数（如 `"标题行数：";"39";`），之后是元数据
- 数据：时间序列（trial_time, recording_time, x_center, y_center, zone, area_change, elongation, distance_moved, velocity, ...）
- 单文件：约 9000 行、1-2 MB
- Demo 数据位置：`/home/qiuyangwang/ethoinsight-demo/DemoData/`（11 种范式）

## 4. 关键决策记录

### 4.1 架构演进（3 轮迭代）

| 版本 | 方案 | 被否原因 |
|------|------|---------|
| v1 | 5 个固定 DeerFlow tool（parse/compute/statistics/chart/assess）| 用户指出：格式固定但需求不同，应该用沙箱代码执行 |
| v2 | 纯沙箱代码执行（Agent 每次写全部代码）| 太慢，应该用预制模板节省时间 |
| v3（定稿）| 预制模板 + 3 个 subagent + handoff 交接 | ✅ 当前方案 |

### 4.2 核心架构决策

- **不注册固定 tool**：code-executor 通过 bash 在沙箱执行预制模板，根据用户需求微调代码
- **3 个专业 subagent**：code-executor（代码执行）、data-analyst（领域洞察）、report-writer（报告）
- **严格串行**：code-executor → data-analyst → report-writer，lead agent 协调
- **handoff JSON 交接**：subagent 写 JSON 给 lead agent 看状态，数据 output 文件给下游 agent 读
- **预制模板**：每种实验 setup 一个完整可执行 .py 脚本，参数在顶部便于 LLM 修改
- **兜底**：执行失败 → 读数据前 500 行 → 修代码 → 重试最多 3 次

### 4.3 其他决策

- **Monorepo**：noldus-insight 为主仓库，deerflow fork 作为嵌套 git repo
- **GLM 版本**：GLM-5.1（交接文档为准，之前记忆中的 GLM-4 已更正）
- **Prompt cache**：利用 GLM-5.1 自动前缀缓存，静态内容前置
- **上游同步**：不主动跟进 DeerFlow 更新，只 cherry-pick 需要的 bugfix

## 7. 风险与注意事项

- **DeerFlow 的 harness 边界**：`deerflow.*` 不能 import `app.*`。新 subagent 配置文件放在 `deerflow/subagents/builtins/` 下没问题，但 tool 函数如果放在 `app.tools.*` 则不能被 harness 直接引用（只能通过 config.yaml 反射加载）。当前方案不注册 tool 所以不受影响。
- **uploads.py 中 import ethoinsight**：需要确保 ethoinsight 包安装到了 backend 的 Python 环境中（`uv pip install -e ../../ethoinsight`）
- **subagent 的 tools allowlist**：必须精确指定，否则 subagent 会继承所有工具
- **DeerFlow 的 prompt.py 中 subagent 列表是硬编码的**：需要改为动态生成，否则新 subagent 不会出现在 lead agent 的 system prompt 中
- **EthoVision 文件的 header 行数不一定是 39**：第一行声明行数，parse.py 必须动态读取
- **config.yaml 中的 `ethoinsight` tool_group 不需要了**：v3 方案不注册 tool，config.yaml 只需要 GLM-5.1 模型配置
