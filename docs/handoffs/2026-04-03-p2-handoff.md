### 3.2 metrics.py 核心数据结构

**`compute_paradigm_metrics()` 返回结构**：
```python
{
    "paradigm": "shoaling",
    "per_subject": {
        "Subject 1": {
            "distance_moved": 12345.6,
            "velocity_stats": {"mean": 4.1, "std": 2.3, "max": 15.0, "min": 0.0, "median": 3.5},
            "mean_nnd": 45.2,       # shoaling only

        },
        ...
    },
    "group_summary": {
        "control": {

            ...
        },
        "treatment": {...},
    },
    "timeseries": {

        "nearest_neighbor_distance": DataFrame(trial_time, subject, nnd),
        "group_polarity": DataFrame(trial_time, polarity),
    },
    "metadata": {"n_subjects": 5, "n_files": 5, "duration_s": 300.2, "computed_metrics": [...]},
}
```

### 3.3 charts.py 接口约定

- 所有图表函数接收 `metrics` dict（来自 `compute_paradigm_metrics` 的输出）
- `output_path=None` 时自动保存到 `/tmp/ethoinsight/`
- 返回保存的文件路径字符串
- `significance` 参数预期 P3 的 `statistics.compare_groups()` 返回格式：`{"comparisons": {"metric_name": [{"group1": "a", "group2": "b", "p_value": 0.03}]}}`

### 3.4 Zone 列数据格式

EPM/Open Field 的 zone 列是 **int64 (0/1)**，不是 bool。metrics.py 的 `compute_open_arm_time_ratio` 和 `compute_center_time_ratio` 用 `.mean()` 计算比率，兼容 0/1 和 True/False。

### 3.5 Shoaling 数据关键发现

- 35 个文件 = 5 subject × 7 导出配置（不同列集合）
- 基础文件（`Subject ?.txt` 无括号后缀）= 核心数据：x_center, y_center, distance_moved, velocity
- **没有 heading 列** → `compute_group_polarity` 从连续帧坐标差推导方向角
- 8997 行 × 5 subject，约 30fps，300 秒
- 坐标单位 mm，x ∈ [-50, 295]，y ∈ [-158, 187]

## 4. 关键决策记录

| 决策 | 选择 | 原因 |
|------|------|------|
| IID 计算方式 | 对齐 trial_time → C(n,2) 对欧式距离 | 需要同一时刻的位置对比 |
| Group polarity | atan2(dy,dx) + 5帧滑动平滑 + mean resultant length | 无 heading 列，30fps 太密集需平滑 |
| Zone 列查找 | 正则匹配（`in_zone.*open_arm`） | 不同导出配置列名差异大 |
| Chart style | Okabe-Ito 调色板 + 无上右边框 + 300 DPI | 色盲友好 + 发表级质量 |
| Template 统计跳过 | `hasattr(stats, 'compare_groups')` | stub 模块存在但无函数，ImportError 不触发 |
| NND 输出格式 | 长格式（trial_time, subject, nnd） | 每个 subject 在每个时间点都有不同 NND |
