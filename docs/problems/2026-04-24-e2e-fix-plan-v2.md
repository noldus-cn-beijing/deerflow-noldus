## 2. 需要修改的文件清单（5 个，全在 skill/library 层）

### 改动 1：`packages/agent/skills/custom/ethoinsight/references/paradigm-interpretation.md`

**问题**：整份文件以"正常范围 / 高焦虑 / 低焦虑"三栏绝对阈值表组织，违反 golden case ANSWER 第 41 行"不给定 baseline 或常模"和 CLAUDE.md 第 9 条"组间比较，不用绝对阈值"。

**改法**：保留指标含义说明，**删除所有"正常范围"数值栏**，改成"**组间对比要点**"。

修改前（第 3-8 行示例）：
```
## 高架十字迷宫 (EPM)
| 指标 | 正常范围 (C57BL/6J) | 高焦虑 | 低焦虑 |
|------|---------------------|--------|--------|
| 开臂时间比 | 15-25% | < 10% | > 30% |
```

修改后：
```
## 高架十字迷宫 (EPM)

### 核心指标
- **开臂时间比**: 焦虑水平的经典读出；treatment 组显著低于 control → 焦虑样行为
- **开臂进入比**: 开臂进入次数 / 总进入次数，补充开臂时间比
- **总臂进入次数**: 总活动量，用于排除运动量混杂（不作为焦虑读出）

### 判读原则
- **组间对比为唯一判据**：以统计检验（Mann-Whitney / t-test 视分布）+ 效应量（Cohen's d）为结论基础
- **不使用绝对阈值或常模范围**——行为学数据受品系/日龄/温度/光照等多因素影响，无跨实验通用 baseline
- **结果报告仅含统计量**；行为学解读放 Discussion 段
```

所有 6 个范式（EPM、OFT、Zero Maze、Light-Dark、NOR、Y-maze、Shoaling）都按此模式改写。shoaling 段（第 39-43 行）还要加 golden case notes.md 里的新信息：

```
## 斑马鱼群体行为 (Shoaling)

### 核心指标
- **IID (Inter-Individual Distance)**: 所有鱼两两距离的平均值，反映群体紧密度
- **NND (Nearest Neighbor Distance)**: 每条鱼到最近同伴的距离，对离群个体敏感
- **群体极性 (Polarity)**: R ∈ [0,1]，越高越同向
- **象限分布**: 四象限内的鱼数分布及聚集时间（需 EthoVision 项目启用对应 JS 变量）

### 数据来源说明
- IID / NND / 象限分布在 EthoVision XT 中通常是 **JS Continuous/State 自定义变量**，需研究员在项目里启用对应脚本
- 导出 raw data **不一定包含这些列**——ethoinsight.metrics 会从 X/Y 坐标自行计算作为兜底
- 两种来源的数值可能不同（脚本版有研究员的领域参数，坐标计算版是通用几何）。handoff 应标注数据来源

### 判读原则
- 组间对比为唯一判据，不使用绝对阈值
- IID/Polarity 需确认至少有 2 个 subject 同时追踪（metrics.py 单鱼时返回 `{"applicable": false}`）
```

### 改动 2：`packages/agent/skills/custom/ethoinsight/references/confound-checklist.md`

**问题**：第 6 行"总距离 < 50% 或 > 200% → 焦虑/抑郁指标不可信（可能是运动障碍）"——把 distance_moved 当成焦虑指标的"污染源"，这违反 golden case ANSWER 第 55 行。

**改法**：删除这条，或重写为"**仅作为混杂因素备注，不作为离群或排除依据**"：

```
- **运动量差异的处理方式**（新）: 若某 subject 的 distance_moved 或 velocity 与同组其他个体差异较大：
    - 不作为离群值排除依据（运动量低可能是探索策略差异，不是病理异常）
    - 不作为"焦虑指标不可信"的理由
    - 可作为 Discussion 段的混杂因素候选，提示研究员检查是否有独立的健康/造模依据
```

### 改动 3：`packages/agent/skills/custom/ethoinsight-planning/references/quality-gates.md`

**问题 a**：第 30 行 "某只动物总运动量异常偏高（> 对照组 200%）→ 可能运动亢进" 和 "偏低（< 50%）→ 可能运动障碍"——同上。

**改法**：从"常见 warnings"列表里**删除**这两条。保留"轨迹中断（missing data > 10%）"和"采样频率不一致"。

**问题 b**：Gate 1 样本量检查表格（第 11-15 行）没问题，保留。

**问题 c（新增）**：Gate 1 之后加一个 **Gate 1.5：分组语义核实**

```markdown
## Gate 1.5: 规划阶段 — 分组语义检查

**触发时机**: Gate 1 样本量检查之后，Step 5 输出计划之前

**检查**: 用户提供的 group name 是否为通用 label 但缺少处理描述

**触发条件**:
- group name 属于 `{control, treatment, 对照, 实验, ctrl, trt, 组 1, 组 2}` 等通用 label
- **且** 用户的消息中未同时提到药物名 / 剂量 / 造模类型 / 基因型 / 年龄 / 性别差异等实质性处理信息

**行动**: `ask_clarification`，问：
> "请问 [Treatment/实验组] 对应的具体处理是什么？例如药物剂量、造模类型、基因型差异等。**这会影响 Discussion 段的解读方向——Result 段不受影响**。"

**session 内去重**: 用户回答后，把处理描述写入 handoff_planning.json 的 `group_semantics` 字段，本 session 内不再追问。下次 planning 先读该字段。
```
