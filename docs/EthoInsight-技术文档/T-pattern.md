	简介
T-Pattern，全称为Temporal Pattern，即时间模式。它是由Magnus S. Magnusson教授在1990年代提出的一个概念，用于识别在时间上非随机发生的、重复出现的事件序列。
核心思想：
传统的行为分析可能会统计某个行为发生的频率或总时长，但这会忽略行为在时间上的结构。T-Pattern分析的核心在于，它认为重要的不是单个行为，而是行为之间具有时间关联性的组合。这些行为组合在发生时，其内部事件之间的时间间隔具有统计学上的显著性，不是随机发生的。
一个简单的例子：
在观察婴儿和母亲的互动时，单独看“婴儿笑”和“母亲说话”的频率可能没有特别的意义。但T-Pattern分析可能会发现一个规律：“婴儿笑” → （在1秒内）→ “母亲说话” → （在2秒内）→ “婴儿再次笑” 这个序列会显著地重复出现。这个序列就是一个T-pattern，它揭示了互动中的“对话轮流”结构。

	数据库
目前没有标准的Python库专门用于T-pattern分析。最权威、功能最完整的工具是THEME软件（未公布算法），Python中需要通过组合现有统计库和自定义算法来实现，R语言中有一个名为T-Pattern的包，实用性不强。

	算法
THEME实际算法（基于Magnusson论文）：
	使用精确二项检验但包含连续性校正
	考虑事件持续时间
	使用最优关键间隔搜索
	包含多重比较校正

THEME算法的核心数学公式
根据Magnusson的论文，THEME使用的实际检验公式是：
关键概率计算：
对于事件A和B，在关键间隔 [d1, d2] 内：
P=((T_B+(d_2-d_1))^k×(T-T_B-(d_2-d_1))^(N_A-k))/T^(N_A ) ×(N_A !)/(k!(N_A-k)!)

其中：
	T = 总观察时间
	T_B = 行为B的总持续时间
	N_A = 行为A的发生次数
	k = 在A之后的 [d1, d2] 窗口内B发生的次数
	d1, d2 = 关键间隔的开始和结束时间
连续性校正：
THEME算法考虑了事件的持续时间，实际计算的是：
P_"effective" =P×(1+〖"duration" 〗_B/(d_2-d_1 ))

	数据输入
必需的核心数据字段：
1. 行为事件基本数据
	行为编码：标准化的行为分类（如：攻击、理毛、嗅闻、玩耍等）
	开始时间：精确到秒或毫秒的行为起始时间戳
	结束时间：行为结束的时间戳
	持续时间：结束时间减开始时间（可由软件计算）
2. 个体与实验信息
	个体标识：执行行为的动物ID
	实验条件/分组：不同处理组、环境条件或实验阶段
	会话标识：如果有多轮实验，需要会话ID
推荐的扩展数据字段：
3. 社会互动信息（如研究社会行为）
	行为对象：互动目标个体的ID
	互动类型：发起/回应、主动/被动等
	空间关系：互动时的距离、方位
4. 环境与情境数据
	空间位置：行为发生的位置坐标或区域
	环境条件：光照、温度、资源分布等
	时间标记：相对于实验开始的时间

	数据输出
1. 检测到的T-Pattern详细信息
每个模式的描述：
	模式结构：行为序列的具体组成（如："接近→嗅闻→理毛"）
	出现频率：在整个观察期内该模式出现的次数
	统计显著性：p值、显著性水平标记
	时间特征：模式内部各行为间的时间间隔范围
	持续时间：整个模式从开始到结束的平均时长
模式元数据：
	模式ID编号
	模式复杂度（包含的行为数量）
	首次和最后一次出现的时间点
	在时间轴上的分布特征
2. 统计分析结果
总体统计：
	检测到的模式总数
	简单模式（二元）vs 复杂模式数量
	最长模式的长度（行为数量）
	最显著和最频繁的模式标识
分组比较（如有多个实验条件）：
	不同组别特有的模式
	模式频率的组间差异统计
	模式结构的条件特异性
3. 时间动态分析
时间分布特征：
	模式在不同时间段的出现频率
	模式出现的时间聚类特征
	随时间变化的模式演化
序列特性：
	模式出现的规律性（周期性、随机性）
	模式之间的转换关系
	行为序列的马尔可夫性质
4. 行为生态学解读数据
功能意义分析：
	社会互动模式（dominance、affiliation等）
	个体行为序列的"语法"结构
	环境适应的行为策略模式
个体差异：
	不同个体的特征性行为模式
	个性特质相关的序列特征
	行为灵活性的量化指标
5. 可视化与报告数据
图表数据：
	模式显著性排序图
	行为序列网络图
	时间分布直方图
	组间比较统计图
研究报告数据：
	关键发现的统计摘要
	重要模式的详细描述
	方法学参数记录
	质量控制指标
