The Solution

多智能体协作的 AI 分析引擎

EthoInsight 不是又一个"套壳 ChatGPT"。它是一个由 5 个专业 AI 智能体协作完成复杂分析任务的系统工程 —— 每个智能体只做一件事，做到极致，通过结构化 Handoff 协议传递 hard fact，而非 LLM 自由发挥。

🧠

Lead Agent（调度中枢）

理解研究员的中文意图，识别 EthoVision XT 19 种模板类型，生成结构化分析计划（metric_plan.json），派遣子智能体按序执行。

⚙️

Code Executor（计算引擎）

基于声明式 Catalog（YAML SSOT）调用 ethoinsight 分析库，自动完成数据解析、指标计算、图表生成。所有计算确定性可复现。

🔬

Data Analyst（洞察引擎）

审核统计方法选择、排查混杂因素（样本量不均/体重差异/时辰）、发现数据中研究员可能忽略的洞察。三层判断：统计 → 领域 → 质量。

📝

Report Writer（报告引擎）

读取分析结果 + 洞察，生成结构化 APA 格式报告：Results（统计 + 效应量 + 图表引用）+ Discussion（解读 + 文献对比 + 局限性）。

📚

Knowledge Assistant（知识引擎）

回答范式、术语、方法论追问。跨会话记忆：用户上次 EPM 焦虑、这次 OFT 也焦虑 → 汇聚证据链，提升结论置信度。

Architecture

声明式知识驱动 · 智能在节点内 · 约束在节点间

区别于传统"LLM 自由发挥"的 Agent 系统，EthoInsight 采用声明式编排哲学：所有可枚举的知识（范式定义、指标公式、图表规则） 下沉到 YAML Catalog 作为单一真相源（SSOT），LLM 只负责"不可枚举的"——意图理解和领域推理。

Cutting-Edge Technology

技术栈与创新亮点

EthoInsight 站在多条前沿技术路线的交汇点上。以下每一项单独拿出来都是学术界/工业界正在活跃探索的方向， 而我们将其集成到一个可工作的产品中。

🔬 SkillOpt：用强化学习自动优化 AI 指令

传统的 AI Agent 开发中，Skill/Prompt 靠人工反复试错调整——成本极高且无收敛保证。 我们引入微软研究院的 SkillOpt 方法论，将 Skill 优化建模为可量化的优化问题：

关键创新：自定义 EthoInsight EnvAdapter，让 SkillOpt 的训练循环直接驱动我们的多智能体流水线。 每个范式独立优化，产出经过 benchmark 验证的 best_skill.md。优化后的 Skill 再驱动 Agent 生成高质量 SFT 训练数据， 最终微调 Qwen3-30B-A3B MoE 模型 —— 形成"专家知识 → Skill 优化 → SFT 数据 → 模型内化"的完整飞轮。

🧬 多智能体编排（Multi-Agent Orchestration）

声明式编排

所有可枚举知识（指标、图表、统计规则）下沉到 YAML Catalog，作为单一真相源。LLM 只处理不可枚举的——意图理解和领域推理。杜绝"LLM 试错"。

结构化 Handoff 协议

子智能体之间通过 JSON Schema 约束的 Handoff 文件传递 hard fact，不靠 LLM 自由文本"口口相传"。每个字段有明确契约。

Guardrail 中间件链

9 层 Guardrail Provider 组成的防御纵深：范式验证 → 数据质量门禁 → 派遣授权 → 参数审计 → 越权防御。每层独立可测试。

跨会话 Memory

基于 DeerFlow Tier 4 持久化架构，per-user 隔离。实验摘要自动存储，下次同范式分析时注入历史 effect size 作为先验参考。

Vision

最终理想：AI 成为行为学家的"第二大脑"

这不是"AI 替代研究员"，而是"AI 成为研究员的分析伙伴"。 研究员专注于实验设计和科学判断，AI 处理繁琐的数据处理、统计选择和报告撰写 —— 同时发现研究员可能忽略的洞察。

三阶段愿景

Phase 1

数据分析助手（v0.1）

上传数据 → 自动分析 → APA 报告。替代研究员 80% 的手动数据处理和统计工作。

Phase 2

实验设计伙伴（v0.2）

实验前：基于历史数据推荐样本量、分组方案、检测指标。实验中：实时数据质量监控。实验后：跨范式证据链汇聚。

Phase 3

科学发现加速器（v1.0+）

主动发现："你上周 EPM 的开放臂时间比例和这周 OFT 的中心区时间呈强相关，建议进一步分析。"跨实验、跨范式、跨模态的知识融合。
