# 预填反诘话术模板

## 正例 1：OFT 三列自定义分析区（标准主反问）

```
⚠️ 开始分析前需确认：

1. EV19 模板：OpenFieldRectangle（已识别，确认即可）
2. 你的数据有 3 列是自定义分析区。基于各列取值分布，我的初步理解是：

┌──────────────┬────────────────┬──────────────────┐
│ 你的列名      │ 取值=1 时间占比 │ 我的理解（请确认） │
├──────────────┼────────────────┼──────────────────┤
│ 中心区        │ 18%            │ → 中心区 center   │
│ 边缘区        │ 82%            │ → 边缘区 border   │
│ 边缘区到中心区 │ （非 0/1）      │ → 疑似距离列，忽略 │
└──────────────┴────────────────┴──────────────────┘

OFT 分析的区类型有：中心区 / 边缘区 / 角落 / 其它(忽略)。

以上理解对吗？如有错误请告诉我正确对应。
分组已识别为 KK/MM/MN/SS/TT，无需填。
```

关键要素：
- 模板确认在前（"已识别，确认即可"）
- 预填列标"请确认"
- 证据（取值占比）与预填并列，让用户据证据核对
- 给否决口子
- 分组已识别 → 不重复问（避免冗余）
- 合法概念菜单（OFT 分析的区类型有...）

## 正例 2：用户否决后二次确认

```
确认一下：你把「边缘区」标为 中心区(center)。这列取值=1 占 82%。
（中心区通常是动物较少停留的区，仅供核对。）这是本意吗？回复"是"或更正。
```

关键要素：
- 引用证据（82%）
- 领域知识提示（非强制，仅供核对）
- 二元选择（"是"或更正）

## 反例 1：字面猜测

```
❌ 列名叫"中心区"，所以应该是 in_zone_center。——请确认，我直接用了这个名字。
```

问题：预填来自字面，不是证据+catalog。正确做法：引用证据（占比 18%）+ catalog 概念菜单。

## 反例 2：无预填直接问

```
❌ 你的数据有 3 列不认识的：中心区、边缘区、边缘区到中心区。请问这些列分别是什么？
```

问题：用户不知道 catalog 概念名（in_zone_center、in_zone_border），无法回答。正确做法：预填+请确认。

## 反例 3：未展示证据

```
❌ 我的理解是：中心区→center，边缘区→border，边缘区到中心区→ignore。对吗？
```

问题：没有展示证据（取值占比），用户无法核查预填是否合理。正确做法：证据与预填并列。
